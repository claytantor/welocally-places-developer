What is a Welocally Place?

Welocally Places For Developers is a set of jQuery based javascript widgets that make it easy to create highly styled customized maps using Google Maps API v3. Its perfect for creating business directory websites, travel sites, real estate guides or any website where you want to go beyond the look and feel of Google Maps and showcase real world places.

It is founded around the data of places. Simply put, a place is all the relevant facts about a real world location, like its name, location, phone number, and website. This data is then used to set the map location and create place cards or marker maps.

Our primary goal is to combine a number of aspects of the Google Maps API into a single easy to use set of components, helping developers build beautiful and unique hyper-local websites with ease. Need more help? Goto our support site for in depth assistance with this product.

Place Widget

The place widget creates a pin card for a specific place. It perfect for showcasing a specific location. Of course all of the Google Maps Capabilities work also, so users can use street view to get curb appeal quickly.

Multi Places Widget

The multi places widget will automatically scale to the correct view bounds to include all the place markers. When a user selects a specific place marker details like website or phone number and driving directions become visible and the map will center. This map is great for creating a set of places with a specific category, such as "Shopping" or a travel map.

Place Widget Configuration

The place widget configuration options are limited to marker and map styling. Most of the styling options reside in the CSS for the widget (see Place Styling tab for more info)

imagePath: The path to the markers index image, photoshop files included for each theme in examples directory.

styles: uses json styles from Google Map Styles Wizard for themeing your map.

Multi Widget Configuration

showLetters: Index the markers automatically, you cant have more than 25 (A-Z). Turning this off uses the same image marker for all items.

showSelection: Will turn on a selectable list of items, essentailly a place list.

overrideSelectableStyle: This is the style override for seletable list elements. Used for one offs.

styles: uses json styles from Google Map Styles Wizard for themeing your map.

The HTML Header

The HTML is used to configure the scripts and the styling that will be used for the map widget. It also is used to include the dependecies for the widget. Which are:

  • jQuery 1.7.1
  • jQuery UI 1.8.18
  • Google Maps V3
  • Google Web Fonts (Optional)

Styling For IE

We support Chome 18, Firefox 5,6,7 and Safari 5.1. While we do not reccomend supporting Internet Explorer, we understand it is a necessity in some situations that can not be avoided. Our maps have been tested and work in IE6, IE7, IE8 and IE9 but do not look as good or perform as well as other browsers. Whe have included IE styling for our basic theme in the Welocally examples styles as an example on how you can support IE. If you want to make a custom theme that supports IE it should work as long as it passes all tests using the CSS Validation Service

Every aspect of the place map is stylable through css, primarily with the file wl_places_place.css, with the exception of the map itself which is styled by using json styles from Google Map Styles Wizard, and the markers image (See Multi Widget Configuration).

Googe Docs Cloud Place

Googe Apps Cloud Places are cloud enabled places that can fetched remotely from your google apps spreadsheet. Just create a single worksheet spreadsheet with your places and all your sites will show the same information. Use google docs as your cloud places database!

To make your google docs spreadsheet the cloud database for you places you must publish your spreadsheet publically, and publish the places you want available as a single worksheet based spreadsheet. Make sure that updates are made saved so all maps pointing to this spreadsheet will be automatically updated.

Once you have published your spreadsheet it now acts as the cloud database for all your places, for a single place card just specify which row you want shown.

We have included a CSV file that you should be able to use as a template for your own google docs places spreadsheet. Download it and use it to put your own places in the cloud.

Google Docs Cloud Places

The same approach that is used to load a single place in the previous example is used to create a multi place map, except that instead of just loading a single place the multi place map will attempt to load all places in the spreadsheet.

Using the cloud can be helpful for syndicating and managing places you may use on multiple sites from one data source, without databases or complicated APIs. We also have a simple cloud service for larger applications we are working on now. If you are interested in learning how our cloud places products can help you with your next location based application, please contact us.

Batch Geocoder

We found a geocoder for google docs. We dont want to distribute it because we didnt write it, but the author has made publically available, so we are providing a link to the script. Its super easy to use. Just add the code to your google spreadsheet using Tools->Script manager... and the run onLoad. A new menu item will appear, Geocode. Select the address fields and the lat and long columns, and it will geocode all the rows in your spreadsheet.
